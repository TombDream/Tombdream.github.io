Android 开发中的那些坑

2026 年 5 月 8 日 · 10 分钟阅读

做 Android 开发这些年，踩过无数坑。有些坑是 Android 系统本身的，有些是第三方库的，还有些是自己挖的。这里记录一些最有代表性的。

1. WebView 的坑

WebView 是 Android 中最容易出问题的组件之一。不同系统版本、不同厂商的 WebView 实现差异很大。

// ❌ 错误做法：直接使用 WebView
WebView webView = new WebView(context);
// 内存泄漏风险！

// ✅ 正确做法：使用 Application Context
WebView webView = new WebView(getApplicationContext());

另一个常见问题是 内存泄漏。WebView 持有 Activity 引用时，会导致 Activity 无法被回收。

2. 混淆配置的坑

开启混淆后，很多运行时的问题会冒出来。特别是反射调用的类、序列化对象、JNI 相关代码，都需要额外配置。

# 保留 WebView 相关
-keepclassmembers class * extends android.webkit.WebView {
    public *;
}

# 保留序列化对象
-keepclassmembers class * implements java.io.Serializable {
    static final long serialVersionUID;
    private static final java.io.ObjectStreamField[] serialPersistentFields;
    !static !transient ;
    private void writeObject(java.io.ObjectOutputStream);
    private void readObject(java.io.ObjectInputStream);
    java.lang.Object writeReplace();
    java.lang.Object readResolve();
}

3. 主线程阻塞

Android 要求 UI 操作必须在主线程执行，但很多人会不小心在主线程做网络请求或数据库操作。

永远不要在主线程执行超过 5ms 的操作。用 StrictMode 检测问题。

4. 后台服务限制

从 Android 8.0（API 26）开始，后台服务受到严格限制。需要用 WorkManager 或 ForegroundService 替代。

5. APK 反编译保护

如果应用涉及敏感逻辑，一定要做反编译保护。仅靠混淆是不够的。

  • 使用 ProGuard 或 R8 开启混淆
  • 关键逻辑用 NDK（C/C++）实现
  • 使用防二次打包检测
  • 敏感字符串加密存储，运行时解密

总结

Android 开发的坑还有很多，比如权限适配、深色模式、折叠屏适配、车载 HUD 显示等等。关键是多测试、多记录、多分享。
